### 1. Course Description

This course explores basic to advanced database engineering concepts beyond traditional relational models, covering modern architectures such as NoSQL, NewSQL, distributed databases, and big data technologies. It covers advanced indexing, query optimization, transaction management, and scalability techniques. This course expects to design and implement efficient real scenario database by providing essential knowledge and skills related to modern database and its application through case studies, group works, readings, and practical knowledge.

### 2. General Objectives

This course is designed to equip students with an overview of database, database management system, distributed database and its applications in various domains. It provides sufficient knowledge of analyzing advanced database architectures, including distributed and cloud-based systems, apply indexing and query optimization techniques for high-performance databases, Design and implement NoSQL databases (document, key-value, column-family, graph), Evaluate trade-offs in distributed databases (CAP theorem, consistency models), Utilize big data processing frameworks (Hadoop, Spark) in database applications, Develop scalable, fault-tolerant database solutions for real-world scenarios.

### 5. Case Studies

Case Studies shall cover the following works/studies with suitable tools like MySQL, PostgreSQL, PostGIS, MongoDB, Python, Grafana, AWS RDS, Azure Cosmos DB etc: SQL & Advanced SQL operations, Query Optimization and Execution Plans, CRUD Operations & Indexing, Distributed Transactions (2PC Simulation), Time Series Data, Spatial Data, Security issues in distributed databases.

### 6. Evaluation System and Students' Responsibilities

**Evaluation System**
In addition to the formal exam(s) conducted by the Office of the Controller of Examination of Pokhara University, the internal evaluation of a student may consist of class attendance, class participation, case studies, research paper review, assignments, presentations, written exams, etc. The tabular presentation of the evaluation system is as follows.

| **Total External**         | **40** | **Total Internal**                  | **60** |
| **Full Marks 40+60 = 100** |        |                                     |        |

**Students' Responsibilities:**
Each student must secure at least 60% marks in the internal evaluation with 80% attendance in the class to appear in the Semester End Examination. Failing to obtain such a score will be given NOT QUALIFIED (NQ) and the student will not be eligible to appear in the End-Term examinations. Students are advised to attend all the classes and complete all the assignments within the specified time period. If a student does not attend the class(es), it is his/her sole responsibility to cover the topic(s) taught during the period. If a student fails to attend a formal exam, quiz, test, etc. there will not be any provision for a re-exam.
